Grid

Introduction

Genius Kitchen uses a responsive, mobile first fluid grid system based on the Bootstrap project that scales between 8 and 28 columns based on browser width.

The GK grid has two distinct behaviors as the browser window grows. All breakpoints start with columns of 21px, surrounded by gutters of 14px and outside site gutters of 14px. On the smallest (XS) breakpoint, the site grid stretches to maintain the overall percentage of the screen. However, on larger breakpoints the columns & gutters remain at 21px and 14, respectively.

Breakpoints

Breakpoint Name Target Device Lower Limit Upper Limit Columns LESS variable prefix Effective site width
X-Small1 Mobile 0 767px 8 @screen-xs- 294px
Small Tablets 768px 1024px 21 @screen-sm- 749px
Medium Desktop 1025px (none) 28 @screen-md- 995px

1 Since the lower limit for the smallest breakpoint, all designs & styles are based off a 320px baseline. To reference this baseline, please use the @screen-xs-baseline

Media Queries

Like all responsive sites, Genius Kitchen uses media queries to change layouts between pre-set breakpoints. When targeting breakpoints using LESS, please use the appropriate variables. There is no need to use a media query for the smallest breakpoint, since Genius Kitchen is mobile-first. However, if you want to restrict to ONLY the XS breakpoint, you can use @screen-xs-max as the max-width.

/* Tablets (768 – 1024px) */
@media  (min-width:  @screen-sm-min)  { … }

/* Desktop (1025 – ∞) */
@media  (min-width:  @screen-md-min)  { … }

There is a preferred, shorthand method for media queries using LESS mixins, as detailed below:

@media @xs-only { ... } // covers the extra small screen size only
@media @sm      { … } // covers screen sizes with a minimum width of 768px
@media @sm-only { … } // covers the small screen size only
@media @md      { … } // covers screen sizes with a minimum width of 1025px
@media @md-only { … } // covers the medium screen size only

Grid Variables

GK syling is done using LESS, a CSS pre-processor in order to streamline styling efforts and more easily reuse styles. Please see the complete LESS documentation for more info. Whenever feasable, in order to keep GK responsive, relative widths (percentages) are used for elements.

Spinners

The loading spinner is used when dynamically loading or updating portions of a page to indicate an action is being taken.

Basic Spinner

Loading… 
<div class="gk-spinner">
        <i class="icon-gk-loading"></i>
        <span>Loading…</span>
    </div>

Inline Spinner

Sometimes, due to vertical space constraints, an inline spinner is needed. The markup is identical to the basic spinner, with the addition of the .ib class to the .gk-spinner element.

Loading… 
<div class="gk-spinner ib">
    <i class="icon-gk-loading"></i>
    <span>Loading…</span>
</div>

Tile Spinner

The tile spinner (as the name implies) is used on pages that contain tiles. Since it does not sit inside any kind of container, it is larger in order to have more prominence. The tile loading spinner should not be used inside visible containers (such as inside a tile or card) or on a white background.

Loading…

<div class="tile-message loader">
    <div class="details">
        <h2 class="title">Loading…</h2>
        <div class="cta">
            <i class="icon-fdc-loading spin"></i>
        </div>
    </div>
</div>

Buttons with Text & Icon

Buttons with icons use flexbox to center the text and icons. These buttons should not use pseudo elements (:before or :after) to style the icon because some browsers, like Internet Explorer, do not position pseudo elements well with flexbox.

* Notice the usage of <span> tags around the text of these buttons. The <span> is used as a block container which works well with flexbox on the button.

</button>

Caret Mixin

The bubble caret mixin provides great flexibility for attaching a caret to a block-level element.

.bubble-caret(@v-pos; @h-pos; @dist; @size; @caret-color; @border-width);
Example 1
Example 2
Example 3
.bubble-caret(top; left; 75%; 5px; @gk-black; 0);            // ex. 1
.bubble-caret(bottom; right; 20px; 20px; @pumpkin-dark; 0);  // ex. 2
.bubble-caret(bottom; left; 20px; 30px; @turquoise; 3px);    // ex. 3

// parameters[default]

// @v-pos: [top]/bottom   : Attach to top of element
// @h-pos: [left]/right   : Which direction to point
// @dist:  [30px]         : distance from @h-pos edge
// @size:  [20px]         : width of caret (height is 1/2 width)
// @caret-color: [@black] : duh
// @border-width: [1px]   : if the object attaching to has a border

Hover States

Because the carets are attached to eleemnts via border properties, in order to style the caret for hover or other states, you must target the border side OPPOSITE the attachment. So, for example, if you have a caret on the top of an element, you would style the border-bottom and vice-versa.

button { // ex. 1
    .bubble-caret(top; left; 28px; 20px; @turquoise; 0);

    &:hover:after{
        // caret attached to top of element, so style border-bottom
        border-bottom-color: @berry-mid;
    }
}
button { // ex. 2
    .bubble-caret(bottom; left; 28px; 20px; @turquoise; 0);

    &:hover:after{
        // caret attached to bottom of element, so style border-top
        border-top-color: @berry-mid;
    }
}

Code Documentation

This section contains tips, tricks, and best practices for programming HTML, LESS, and JavaScript on the Genius Kitchen website.

Disabling Hover States

Because iOS devices treat the hover state as a separate click, any clickable element that has a :hover pseudoclass associated with it creates a bizarre user experience: the first click triggers the hover state and the second click triggers the action associated with the anchor tag. This is an unacceptable UX, so every single :hover pseudoclass needs disable itself for iOS devices. A globally available convenience variable named @no-touch has been created in our LESS and below is an example of its usage.

.example-class {
    color: @berry-mid;

    @{no-touch} &:hover {
        text-decoration: underline;
    }
}

NOTE: The @no-touch variable is within curly braces (interpolation).

Spacing Mixin

In order to improve flexibility with spacing & maintain the multiples of 7 used throughout the site, there is a mixin which can easily create a variable to be used inside the scope of where it is defined.

.space(x); // where ‘x’ is multiplied by 7 to be represented by…
@space;    //which can be used anywhere inside the same scope as the mixin

The @space variable can then be manipulated with math if needed in order to achieve the desired spacing.

.someElement {
    .space(1); // 7px

    margin: @space (@space * 2); // 7px 14px

    .someChildElement {
        margin: @space / 2; // 3.5px
    }
}

IMPORTANT:The @space mixin must not be used in the global scope. If not enclosed inside a selector it will apply globally to all .less files that are imported after the current file.
